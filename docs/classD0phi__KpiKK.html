StatusCode D0phi_KpiKK::execute_rest ( )
virtual

Inherited execute method of the Algorithm.

This function is called for each event.

  1. Create selection charged tracks

    1. Initialise PID and skip if it fails:

      • use probability method

      • use and the three ToF detectors. Since BOSS 7.0.4, ParticleID::useTofCorr() should be used for ToF instead of e.g. useTof1.

      • identify only pions and kaons

      • use

    2. Write Particle Identification information of all tracks

    3. Identify type of charged particle and add to related vector: (this package: kaon and pion).

      RecMdcKalTrack (Kalman) is used, but this can be substituted by RecMdcTrack.

      The particle identification first decides whether the track is more likely to have come from a pion or from a kaon.

      A cut is then applied on whether the probability to be a pion (or kaon) is at least fCut_PIDProb_min (see eventual settings in D0phi_KpiKK.txt).

      Finally, the particle ID of the RecMdcKalTrack object is set to pion

      and the (positive) pion is added to the vector of pions.

  2. Write the multiplicities of the selected particles.

  3. Make selection of MC truth particles by looping over the collection of MC particles created in TrackSelector::execute(). See here for a list of PDG codes.

  4. Apply a strict cut on the number of particles: only 2 negative kaons, 1 positive kaon, and 1 positive pion

  5. Loop over MC truth of final decay products.

    1. Only continue if the two kaons are different.

    2. Check topology: only consider that combination which comes from .

    3. Write 'fake' fit results, that is, momenta of the particles reconstructed from MC truth.

  6. Write PID information ("dedx" branch)

  7. Perform Kalman 4-constraint Kalman kinematic fit for all combinations and decide the combinations that results in the 'best' result. The 'best' result is defined as the combination that has the smallest value of: (that is the combination for which the invariant mass of the is closest to ). See D0phi_KpiKK::MeasureForBestFit for the definition of this measure.

    1. Apply max. cut (determined by fCut_PIDChiSq_max).

    2. Write results of the Kalman kinematic fit (all combinations, "fit4c_all").

    3. Decide if this fit is better than the previous

  8. Write results of the Kalman kitematic fit of the best combination ("fit4c_best_*" branches)

◆ fWrite_fit_mc

bool D0phi_KpiKK::fWrite_fit_mc
protected

NTuples are like vectors, but its members do not necessarily have to be of the same type.

In this package, the NTuples are used to store event-by-event information. Its values are then written to the output ROOT file, creating a ROOT TTree. In that sense, each NTuple here represents one TTree within that output ROOT file, and each NTuple::Item represents its leaves. The name of the leaves is determined when calling NTuple::addItem. Note that the NTuple::Items have to be added to the NTuple during the TrackSelector::initialize() step, otherwise they cannot be used as values! This is also the place where you name these variables, so make sure that the structure here is reflected there! Package property that determines whether or not to write particles reconstructed from MC truth.
