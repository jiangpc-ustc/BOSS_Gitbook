StatusCode D0phi_KpiKK::execute_rest ( )
virtual

Inherited execute method of the Algorithm.

This function is called for each event.

  1. Create selection charged tracks

    1. Initialise PID:

      • use probability method

      • identify only pions and kaons

      • use and the three ToF detectors

      • use

    2. Identify type of charged particle and add to related vector: (this package: kaon and pion).

      RecMdcKalTrack (Kalman) is used, but this can be substituted by RecMdcTrack.

      The particle identification first decides whether the track is more likely to have come from a pion or from a kaon.

      A cut is then applied on whether the probability to be a pion (or kaon) is at least fMinPIDProbability (see eventual settings in D0phi_KpiKK.txt).

      If, according to the likelihood method, the particle is still more likely to be a kaon than a pion, the track is rejected.

      Finally, the particle ID of the RecMdcKalTrack object is set to pion

      and the (positive) pion is added to the vector of pions.

    3. Write Particle Identification information

    4. Apply a strict cut on the number of particles: only 2 , 1 , and 1

    5. Write PID information ("dedx" branch)

    6. Perform Kalman 4-constraint Kalman kinematic fit for all combinations and decide the combinations that results in the 'best' result. The 'best' result is defined as the combination that has the smallest value of:

      See D0phi_KpiKK::BestKinematicKalmanFitMeasure for the definition of this measure.

      Apply max. cut (determined by fCut_MaxChiSq).

      Compute the measure for the best Kalman kinematic fit and keep a pointer to this result if better than previous.

      Store results of the Kalman kinematic fit (all combinations).

    7. Write results of the Kalman kitematic fit of the best combination ("fit4c_best" branch)

◆ fDo_fit4c

bool D0phi_KpiKK::fDo_fit4c
protected

NTuples are like vectors, but its members do not necessarily have to be of the same type.

In this package, the NTuples are used to store event-by-event information. Its values are then written to the output ROOT file, creating a ROOT TTree. In that sense, each NTuple here represents one TTree within that output ROOT file, and each NTuple::Item represents its leaves. The name of the leaves is determined when calling NTuple::addItem. Note that the NTuple::Items have to be added to the NTuple during the TrackSelector::initialize() step, otherwise they cannot be used as values! This is also the place where you name these variables, so make sure that the structure here is reflected there! Package property that determines whether or not to perform a 4-constraint Kalman kinematic fit.
