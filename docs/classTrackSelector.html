Detailed Description

Base algorithm that creates a selection of charged and/or neutral tracks.

Constructor for the TrackSelector algorithm.

Here, you should declare properties: give them a name, assign a parameter (data member of TrackSelector), and if required a documentation string. Note that you should define the paramters themselves in the header (TrackSelector/TrackSelector.h) and that you should assign the values in share/jopOptions_TrackSelector.txt. Algorithms should inherit from Gaudi's Algorithm class. See https://dayabay.bnl.gov/dox/GaudiKernel/html/classAlgorithm.html for more details.

  • The "do_<treename>" properties determine whether or not to make a selection of 'good' neutral and/or charged tracks.
  • The "write_<treename>" properties determine whether or not the corresponding TTree/NTuple will be filled. Default values are set in the constructor as well.
  • The "cut_<parameter>" properties determine cuts on certain parameters.

This function encapsulates the addItem procedure for the energy loss branch ("dedx").

This method allows you to perform the same booking method for different types of charged particles (for instance 'all charged particles', kaons, and pions).

  1. "Ngoodhits": Number of good hits (excluding overflow).

  2. "Ntotalhits": Number of good hits (including overflow).

  3. "chie": in case of electron ("number of \f$\sigma\f$ from \f$e^\pm\f$").

  4. "chik": in case of kaon ("number of \f$\sigma\f$ from \f$K^\pm\f$").

  5. "chimu": in case of muon ("number of \f$\sigma\f$ from \f$\mu^\pm\f$").

  6. "chip": in case of proton ("number of \f$\sigma\f$ from \f$p^\pm\f$").

  7. "chipi": in case of pion ("number of \f$\sigma\f$ from \f$\pi^\pm\f$").

  8. "normPH": Normalized pulse height.

  9. "p": Momentum of the track as reconstructed by MDC.

  10. "probPH": Most probable pulse height from truncated mean.

◆ execute()

StatusCode TrackSelector::execute ( )

Inherited execute method of the Algorithm.

This function is called for each event.

  1. Load next event from DST file. For more info see:

    • Namespace EventModel
    • Class EvtRecEvent

    • Type definition EvtRecTrackCol

  2. Create selection charged tracks and write track info:

    Note: the first part of the set of reconstructed tracks are the charged tracks.

    1. Get MDC information

      Todo:
      Check if helixip.pivot(fVertexPoint) doesn't affect fVertexPoint.

    2. Apply primary vertex cuts:

      • $z$ coordinate of the collision point has to be within cut_Vz0_max

      • radius in $xy$ plane has to be less than cut_Vr0_max

      • coordinate of the distance to the interaction point has to be within cut_Rvz0_max

      • distance to the interaction point in plane has to be within cut_Rvxy0_max

    3. Write charged track vertex position info ("charged" branch)

    4. Write dE/dx PID information ("dedx" branch)

    5. Write Time-of-Flight PID information ("tof*" branch)

  3. Create selection of neutral tracks and write track info.

    Note: The second part of the set of reconstructed events consists of the neutral tracks, that is, the photons detected by the EMC (by clustering EMC crystal energies). Each neutral track is paired with each charged track and if their angle is smaller than a certain value (here, 200), the photon track is stored as 'good photon' (added to iGam).

    1. Get EMC information

    2. Apply photon energy cut (set by TrackSelector.cut_PhotonEnergy).

    3. Write neutral track information (if write_neutral is set to true).

    4. Add photon track to vector of neutral tracks (fGoodNeutralTracks).

  4. write event info ("mult" branch)

  5. write event info ("vertex" branch)

  6. Perform derived algoritm as defined in TrackSelector::execute_rest.

◆ initialize()

StatusCode TrackSelector::initialize ( )

(Inherited) initialize step of Algorithm.

This function is called only once in the beginning.

Define and load NTuples here. The NTuples will become the TTrees in the eventual ROOT file, the added NTuple::Items will be the branches of those trees.

  1. "mult": Multiplicities of the total event

    1. "Ntotal": Total number of events per track.

    2. "Ncharge": Number of charged tracks.

    3. "Nneutral": Number of charged tracks.

    4. "NgoodCharged": Number of 'good' charged tracks.

    5. "Nmdc": Number of charged tracks in MDC.

    6. "NgoodNeutral": Number of 'good' neutral tracks.

  2. "vertex": Vertex info

    1. "vx0": coordinate of the collision point.

    2. "vy0": coordinate of the collision point.

    3. "vz0": coordinate of the collision point.

  3. "charged": Charged track info.

    1. "vx": coordinate of the secondary vertex as determined by MDC.

    2. "vy": coordinate of the secondary vertex as determined by MDC.

    3. "vz": coordinate of the secondary vertex as determined by MDC.

    4. "vr": Distance from origin in plane.

    5. "rvxy": Nearest distance to interaction point in plane.

    6. "rvz": Nearest distance to interaction point in direction.

    7. "rvphi": Angle in the plane.

      Todo:
      Get explanation of geometry (angle) definitions in an event.

    8. "phi": Helix angle of the particle.

    9. "p": Momentum of the track.

  4. "neutral": Neutral track info.

    1. "E": Energy of the neutral track as determined by the EM calorimeter.

    2. "x": -coordinate of the neutral track according to the EMC.

    3. "y": -coordinate of the neutral track according to the EMC.

    4. "z": -coordinate of the neutral track according to the EMC.

    5. "phi": -angle of the neutral track according to the EMC.

    6. "theta": -angle of the neutral track according to the EMC.

    7. "time": Time of the neutral track according to the EMC.

      Todo:
      Investigate what this parameter precisely means.

  5. "dedx": energy loss PID branch. See TrackSelector::BookNtupleItemsDedx for more info.

  6. "ToFEC", "ToFIB", and "ToFOB": information from the three Time-of-Flight detectors. See TrackSelector::BookNtupleItemsTof for more info.

  7. "PID": Track PID information.

    • "p": Momentum of the track as reconstructed by MDC.

    • "cost": Theta angle of the track.

    • "chiToFIB": of the inner barrel ToF of the track.

    • "chiToFEC": of the end cap ToF of the track.

    • "chiToFOB": of the outer barrel ToF of the track.

    • "chidEdx": of the energy loss of the identified track.

    • "prob_K": Probability that the track is from a kaon according to the probability method.

    • "prob_e": Probability that the track is from a electron according to the probability method.

    • "prob_mu": Probability that the track is from a muon according to the probability method.

    • "prob_p": Probability that the track is from a proton according to the probability method.

    • "prob_pi": Probability that the track is from a pion according to the probability method.

void TrackSelector::WriteCuts ( )
protected

Write all cuts (name, value, and count of accepted) to a branch called "_cutvalues".

  1. For each cut name, create an NTuple::Item<double> in the map fMap_cuts.
  2. Add the NTuple::Items of fMap_cuts to an NTuple called "_cutvalues".
  3. Write min values to the first entry of "_cutvalues".
  4. Write max values to the second entry of "_cutvalues".
  5. Write the counter values to the third entry of "_cutvalues".

bool TrackSelector::fDo_charged
protected

NTuples are like vectors, but its members do not necessarily have to be of the same type.

In this package, the NTuples are used to store event-by-event information. Its values are then written to the output ROOT file, creating a ROOT TTree. In that sense, each NTuple here represents one TTree within that output ROOT file, and each NTuple::Item represents its leaves. The name of the leaves is determined when calling NTuple::addItem. Note that the NTuple::Items have to be added to the NTuple during the TrackSelector::initialize() step, otherwise they cannot be used as values! This is also the place where you name these variables, so make sure that the structure here is reflected there!
