Detailed Description

General container for BOSS analysis output.

Author
Remco de Boer 雷穆克 (r.e.d.nosp@m.eboe.nosp@m.r@stu.nosp@m.dent.nosp@m.s.uu..nosp@m.nl or remco.nosp@m..de..nosp@m.boer@.nosp@m.ihep.nosp@m..ac.c.nosp@m.n)
Date
October 24th, 2018

The BOSSRootTree class can be used to create several trees from ROOT files generated by the jopOptions_ana (analysis) step of the BOSS framework in the most general way, that is, it only loads the ROOT files and creates an unordered_map of TChains based on (all) trees within those ROOT files. You can create a derived class which maps the map of TChains to a structure with particular names that match the file with which you are working.

Constructor that opens a TFile and unordered_maps its contents.

  1. Make an inventory of ROOT files in input directory and add filenames to the fFileNames list.
  2. If path ends in ".root", treat as single ROOT file and attempt to load it.

  • Abort if path cannot be loaded as ROOT file.

  • Extract directory and file name from input path.

  • Add resulting file name to fFileNames and continue loading as TChain.

  1. Else, treat path as path to directory and make an inventory of ROOT files in this directory.

  • Remove trailing slash / from fDirectoryPath

  • Open file directory using TSystemDirectory.

  • Abort if directory is empty.

  • Loop over list of files in directory

    and add file name if file exists and is no zombie.

  1. Execute LoadChains now that the fFileNames list has been loaded.

◆ LoadChains()

void BOSSOutputLoader::LoadChains ( bool  print = true)
protected

Create a list of TChains based on the files in the directory you are loading.

If the ROOT TFile could be successfully loaded, this method will generate a std::unordered_map of pointers to all TChains in the main folder of the ROOT file. The first file in the directory is used to make an inventory of all TTrees within the file.

  1. Check whether there are files in the fFileNames list.
  2. Load first ROOT file fFileNames list.
  3. Go through keys in the file and add create ChainLoader if the key is a TTree.

  • Attempt to cast loaded TObject to TKey.

  • Read TKey and attempt to cast to TTree.

  • Create ChainLoader object (with underlying TChain) from the loaded TTree.

  • Loop over file names and at them to the ChainLoader object.

  • Book branches using ChainLoader::BookAddresses.

  1. Print terminal output

◆ PrintCuts()

void BOSSOutputLoader::PrintCuts ( )

  1. Search for a TChain called "_cutvalues" and return if it doesn't exist or if it is empty.

  2. Use entry 0 to get the names of the cut parameters. We use a sorted map so that the keys are sorted automatically. At the same time, the maximum number of characters in all of the cut names is also determined.

  3. If "_cutvalues" contains only 1 entry, only print a list of cut paramters and their values. Entry 0 of the "_cutvalues" tree is considered to be that value. (This is for backward compatibility with output of the older versions of TrackSelector.)

  4. If "_cutvalues" contains 3 entries, consider entry 0 to be the min value, entry 1 to be the max, and entry 2 to be count (the number of events or tracks that passed the cut).

    1. Declare remaining two vectors for max and count.

    2. Get values from entry 1 (max).

    3. Get values from entry 2 (count).

  5. And print loaded values as a table: one row per parameters.

    1. Print table header with four columns.

    2. Column 1: cut name.

    3. Column 2: minimum, if available.

    4. Column 3: maximum, if available.

    5. Column 4: counter, if available.
