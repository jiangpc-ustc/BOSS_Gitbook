Detailed Description

This simple object encapsulates a boolean job option.

Options like these are usually used as switches in your algorithm that can be set to true or false in the job options. The constructor only needs a name for this switch. It is convention to start this name with:

  • "do_" if this is to determine whether or not to perform a certain loop;
  • "write_" if this determines whether or not to write a certain TTree/NTuple.
