Analyse events.

Based on the original RhopiAlg package, but now making use of the TrackSelector base algorithm.

Here, you should declare properties: give them a name, assign a parameter (data member of rhopi_pipigg), and if required a documentation string. Note that you should define the paramters themselves in the header (rhopi_pipigg/rhopi_pipigg.h) and that you should assign the values in share/jopOptions_rhopi_pipigg.txt.

  • The "do_<treename>" properties determine whether or not the corresponding TTree/NTuple will be filled. Default values are set in the constructor as well.
  • The "cut_<parameter>" properties determine cuts on certain parameters.

◆ execute_rest()

StatusCode rhopi_pipigg::execute_rest ( )
virtual

Inherited execute method of the Algorithm.

This function is called for each event.

  1. Create selection charged tracks

    1. Initialise PID and skip if it fails:

      • use probability method

      • use and the three ToF detectors

      • identify only pions

      • use

    2. Write Particle Identification information of all tracks

    3. Identify type of charged particle and add to related vector: (this package: only pions).

      RecMdcKalTrack (Kalman) is used, but this can be substituted by RecMdcTrack.

      A cut is then applied on whether the probability to be a pion (or kaon) is at least fCut_MinPIDProb (see eventual settings in rhopi_pipigg.txt).

      If, according to the likelihood method, the particle is still more likely to be a kaon than a pion, the track is rejected.

      Finally, the particle ID of the RecMdcKalTrack object is set to pion

      and the pions ( ) are added to the respective vector of pions.

  2. Create selection neutral tracks (photons)

    1. Get EM calorimeter info

    2. Find angle differences with nearest charged pion

    3. Apply angle cut

    4. Write photon info ("photon" branch)

    5. Apply photon cuts (energy cut has already been applied in TrackSelector)

    6. Add photon track to vector for gammas

  3. Apply a cut on the number of particles: only events with more than 2 photons

  4. Write PID information ("dedx_pi" branch)

  5. Perform Kalman 4-constraint Kalman kinematic fit for all combinations

    Apply max. cut (determined by fCut_MaxPIDChiSq).

    Compute the measure for the best Kalman kinematic fit and keep a pointer to this result if better than previous.

    Write results of the Kalman kinematic fit (all combinations, "fit4c_all").

  6. Write results of the Kalman kitematic fit of the best combination ("fit4c_best" branch)

  7. Apply a cut on the number of particles: only events with one and one

  8. Perform Kalman 5-constraint Kalman kinematic fit for all combinations

    Apply max. cut (determined by fCut_MaxPIDChiSq).

    Compute the measure for the best Kalman kinematic fit and keep a pointer to this result if better than previous.

    Write results of the Kalman kinematic fit (all combinations, "fit5c_all").

  9. Write results of the Kalman kitematic fit of the best combination ("fit5c_best" branch)

◆ fDo_mult_select

bool rhopi_pipigg::fDo_mult_select
protected

NTuples are like vectors, but its members do not necessarily have to be of the same type.

In this package, the NTuples are used to store event-by-event information. Its values are then written to the output ROOT file, creating a ROOT TTree. In that sense, each NTuple here represents one TTree within that output ROOT file, and each NTuple::Item represents its leaves. The name of the leaves is determined when calling NTuple::addItem. Note that the NTuple::Items have to be added to the NTuple during the TrackSelector::initialize() step, otherwise they cannot be used as values! This is also the place where you name these variables, so make sure that the structure here is reflected there! Package property that determines whether or not to write the multiplicities of the selected particles.
