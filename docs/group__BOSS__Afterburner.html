Detailed Description

Classes, namespaces, and functions for the BOSS Afterburner.

The core of these classes is the BOSSOutputLoader class. This class allows you to automatically load a set of ROOT files that have been procuded during the event selection in BOSS. Supporting functions, parameters and other classes are to be found in for instance the CommonFunctions.h header file.

◆ AnalyseBOSSOutput()

void AnalyseBOSSOutput ( )

Main function used when compiling and executing in ROOT.

To investigate the contents of the ROOT file, you first need to know which TTrees and branches it contains. If you simply construct the BOSSOutputLoader by giving it a file name, all TTrees will be loaded automatically as well as addresses for each of their branches. Five the constructer true as its second argument, and the names of these TTrees, their branches, and the types of these branches (behind the slash / after the name) will be printed to the terminal. Do this if your macro throws an exception, because this probably means that you use the wrong names for the trees and or the branches further on in the macro.

◆ BOSSOutputLoader()

BOSSOutputLoader::BOSSOutputLoader ( const char *  directoryPath,
bool  print = true 
)

Constructor that opens a TFile and unordered_maps its contents.

  1. Make an inventory of ROOT files in input directory and add filenames to the fFileNames list.
  2. If path ends in ".root", treat as single ROOT file and attempt to load it.

  • Abort if path cannot be loaded as ROOT file.

  • Extract directory and file name from input path.

  • Add resulting file name to fFileNames and continue loading as TChain.

  1. Else, treat path as path to directory and make an inventory of ROOT files in this directory.

  • Remove trailing slash / from fDirectoryPath

  • Open file directory using TSystemDirectory.

  • Abort if directory is empty.

  • Loop over list of files in directory

    and add file name if file exists and is no zombie.

  1. Execute LoadChains now that the fFileNames list has been loaded.

The mean of the two Gaussian is in both cases taken to be the mass of the particle to be reconstructed. For a pure particle signal, that is, without backround and without a physical particle width, the width of the two Gaussians characterises the resolution of the detector.
